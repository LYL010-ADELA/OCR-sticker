# 封口贴检测逻辑说明

## 一、整体流程（`process_row`）

每个订单有 1~4 张图片（同一商品不同角度），按以下四个阶段处理：

```
订单（1~4 张图片）
 │
 ├─ Phase 1  遍历所有图片，每张都 OCR + 检测包装盒
 │           筛选"正向完整背面照片"候选，同时满足：
 │             ✓ OCR 识别到"扫码即领"（确认是背面）
 │             ✓ 包装盒检测成功（非 fallback）
 │             ✓ 盒子面积 / 图片面积 ≥ 20%（BOX_FRONTAL_MIN_RATIO）
 │           无候选 → 是否规范粘贴=0，直接返回
 │           水印仅从第一张图提取（时间 / 地点）
 │
 ├─ Phase 2  取盒子占比最大的候选图作为唯一检测对象
 │
 ├─ Phase 3  单贴检测（短路逻辑，不合规立即返回，跳过后续）
 │             Step 1  位置验证（先做）
 │                       → 不合规（position_valid=0）→ 是否规范粘贴=0，stop
 │             Step 2  角度验证（位置通过后执行）
 │                       → 偏角超阈值（position_valid=3）→ 是否规范粘贴=0，stop
 │             Step 3  平铺检测（位置+角度均通过后执行）
 │                       → 平铺错误（position_valid=2）→ 是否规范粘贴=0，stop
 │             三步均通过（position_valid=1）→ 进入 Phase 4
 │
 └─ Phase 4  双贴检测（单贴全部合规后才进入）
               dual_code=2（两张扫码贴）→ 是否规范粘贴=0，stop
               has_auth=True
                 → 盒子内找到 Auth 贴纸 → 验证位置
                     → 不合规 → 是否规范粘贴=0，stop
                     → 合规   → 是否规范粘贴=1 ✓
                 → 盒子内未找到 Auth 贴纸（仅盒外文字触发）
                     → 忽略，视为单贴
               dual_code=0（单贴）→ 是否规范粘贴=1 ✓
```

## 二、单贴检测（`check_sticker_placement`）

Phase 3 的核心函数，三步串行，每步可短路。

```
已知：sticker（"扫码即领"文字框坐标）、包装盒 bbox（box_x/y/w/h）
 │
 ▼ Step 1  位置验证（validate_sticker_position）
           rel_x = (cx - box_x) / box_w
           rel_y = (cy - box_y) / box_h
           ┌─ 0.50 ≤ rel_x ≤ 0.95
           │  0.00 ≤ rel_y ≤ 0.30    两者均满足 → 继续
           └─ 任一不满足             → position_valid=0，立即返回
 │
 ▼ Step 2  角度验证（validate_angle）
           用 cv2.minAreaRect 提取贴纸文字框主轴角度
           delta = sticker_angle - box_angle（正向拍摄时 box_angle ≈ 0°）
           ┌─ |delta| ≤ 10°（STICKER_ANGLE_MAX_DEG）→ 继续
           └─ |delta|  > 10°                        → position_valid=3，立即返回
 │
 ▼ Step 3  平铺错误检测（is_flat_sticker）
           仅检查盒子范围内的 OCR 文字（盒外文字跳过）
           ┌─ 检测到"Authorized Reseller"       → position_valid=2（平铺），返回
           ├─ 检测到远端大字"授权经销商"          → position_valid=2，返回
           └─ 未检测到端片文字                  → position_valid=1 ✓
```

> **短路设计**：前一步不通过时立即返回，后续步骤不执行，节省时间并避免误报。

## 三、角度验证（`validate_angle`）

**业务背景**：门店人员手工贴纸时可能存在轻微歪斜，但歪斜过大（贴纸明显倾斜）视为不规范操作。

**实现原理**：

```
贴纸文字框四边形（OCR poly）
        │
        ▼  cv2.minAreaRect
   ((cx, cy), (w, h), rect_angle)
        │
        ▼  normalize_horizontal_angle(rect_angle, w, h)
   sticker_angle：长轴与水平方向夹角，范围 [-90°, 90°]
        │
   delta = sticker_angle - box_angle
        │   （正向完整照片中 box_angle ≈ 0°，即包装盒水平边近似水平）
        ▼
   |delta| ≤ STICKER_ANGLE_MAX_DEG（10°）→ 合规
   |delta|  > STICKER_ANGLE_MAX_DEG     → position_valid=3，不合规
```

**角度归一化说明**：

`cv2.minAreaRect` 返回角度范围为 `(-90°, 0°]`，表示矩形"宽向量"与水平轴夹角。
当矩形宽 < 高时（如竖排文字），"长轴"实为高方向，需 +90° 修正，再统一映射到 `[-90°, 90°]`。

**阈值建议**：

| 场景 | 推荐阈值 |
|------|---------|
| 严格管控 | 6° |
| 默认（当前配置） | **10°** |
| 宽松容忍 | 12° ~ 15° |

> 修改阈值只需改顶部常量：`STICKER_ANGLE_MAX_DEG = 8.0`

**误判处理**：
- 若 OCR 多边形解析失败（极少数情况），角度验证自动跳过（`ok=True`），不影响其他步骤。
- 角度计算以"贴纸相对包装盒水平边"为基准，而非绝对水平，对整体倾斜拍摄有一定容忍。

## 四、包装盒检测（`detect_box_bbox`）

用于建立"以包装盒为基准的坐标系"，三级降级策略：

| 优先级 | 方法 | 适用场景 |
|--------|------|---------|
| 1（`edge`） | Canny 边缘 + 形态学膨胀 → 最大矩形轮廓 | 各色包装盒（iPhone / Mac） |
| 2（`bright`） | 亮色区域阈值 → 最大外包矩形 | Apple 白色包装盒 |
| 3（`fallback`） | 返回整图区域 | 检测失败时保底 |

- 检测在缩放到 1200px 的小图上进行（速度快），坐标再按比例还原到原图。
- Phase 1 筛选候选图时，`fallback` 的图片会被直接排除（不视为"正向完整照片"）。

## 五、位置规范定义

```
包装盒坐标系（左上角原点）：

    0%           50%    95%
    ├────────────┼──────┤  ← 0%
    │            │ ✓合规│
    │            │ 区域 │  ← 30%
    │            ├──────┤
    │            │      │
    │  包装盒正面 │      │
    │            │      │
    │            │      │
    └────────────┴──────┘  ← 100%

rel_x = (贴纸中心像素X - 盒子左边界) / 盒子宽度
rel_y = (贴纸中心像素Y - 盒子上边界) / 盒子高度

合规条件（"扫码即领"贴纸）：
  0.50 ≤ rel_x ≤ 0.95   （右侧 50%~95% 宽度）
  0.00 ≤ rel_y ≤ 0.30   （顶部 0%~30% 高度）

合规条件（"Apple授权专营店"贴纸，仅双贴场景）：
  0.50 ≤ rel_x ≤ 0.95   （右侧 50%~95% 宽度）
  0.70 ≤ rel_y ≤ 1.00   （底部 30%，即从上往下 70%~100%）
```

"贴纸中心"指 OCR 识别到的**文字框的几何中心**。
坐标已从 OCR 缩放图（最大 2000px）**换算回原图尺寸**后再计算。

## 六、平铺错误检测（`is_flat_sticker`）

**问题背景**：贴纸是一张长条，正确贴法是让主体（QR + 积分文字）在盒面、端片（"Authorized Reseller"区域）绕到侧面封口。如果整条平铺在一个面，端片不起封口作用。

**检测信号（语义优先）**：

| 信号 | 说明 | 可靠性 |
|------|------|--------|
| OCR 识别到"Authorized Reseller" | 仅出现在贴纸端片，包装盒内容绝不含此词 | ★★★ 高 |
| OCR 识别到远端大字"授权经销商"（距锚点 > 5×文字高，且非贴近 QR 码区域） | 区分端片大字与 Apple Logo 内的小字 | ★★ 中 |

**盒内过滤**：遍历 OCR 文字时先判断中心点是否在包装盒范围内，盒外文字（背景、桌面等）直接跳过，避免误报。

> **为何不用文字簇宽高比？**
> 包装盒正面有大量文字（型号、规格、条形码），与贴纸处于同一水平带时会被混入簇，
> 导致簇宽度虚高（实测可达 9.9:1），造成大量误报。
> 改用语义信号后，仅当真正可见端片时才触发，误报率大幅降低。

## 七、双贴纸检测（`check_dual_sticker_status` + `check_auth_sticker_position`）

适用于"双贴"规范产品（iPhone 等），部分门店需同时贴两种贴纸。

**检测步骤**：
1. 找出单图内所有"扫码即领"文字位置（`find_all_scan_stickers`）
2. 按 Y 坐标聚类：Y 差 > 图片高度 20% 视为两张不同贴纸，统计 `scan_count`
3. 全图检索"Apple授权专营店"/"授权专营店"/"在你身边"关键词，得到 `has_auth`

| `dual_code` | 含义 | 后续动作 |
|-------------|------|---------|
| -1 | 未找到贴纸 | 正常不会出现（已过候选筛选） |
| 0 | 单贴 | 直接合规 ✓ |
| 1 | 疑似双贴 | 调用 `check_auth_sticker_position` 验证第二张位置 |
| 2 | 双扫码错误 ✗ | 不合规，stop |

**Auth 贴纸位置验证（`check_auth_sticker_position`）防误判机制**：
- 使用 `find_all_auth_stickers_in_box` 收集**盒子范围内**所有命中项（过滤盒外印刷文字）
- 遍历所有候选，**任意一个满足位置条件即判合规**（防止"先遇到盒面印刷文字"的误判）
- 盒子内找不到任何候选 → `found=False` → 降级为单贴（`dual_code=0`），合规 ✓

## 八、输出列说明

| 列名 | 类型 | 含义 |
|------|------|------|
| `是否规范粘贴` | 0/1 | **最终判定**：1=合规 ✓，0=不合规 ✗ |
| `封口贴存在` | 0/1 | 是否识别到"扫码即领"文字 |
| `贴纸位置规范` | -1/0/1/2/3 | -1=无贴纸 \| 0=位置异常 \| 1=规范✓ \| 2=平铺错误 \| 3=角度异常 |
| `贴纸相对X` | 0.0~1.0 | 贴纸中心在盒子宽度方向的相对位置 |
| `贴纸相对Y` | 0.0~1.0 | 贴纸中心在盒子高度方向的相对位置 |
| `贴纸角度` | 数值（°） | 贴纸长轴与包装盒水平方向的偏角；空=未计算 |
| `包装盒检测方式` | edge/bright/fallback | OpenCV 检测方案 |
| `位置说明` | 文字 | 详细说明（含坐标、偏角、平铺信息） |
| `双贴纸状态` | -1/0/1/2 | 见第七节 |
| `双贴纸说明` | 文字 | 详细说明 |
| `时间` | 文字 | 水印中的拍摄时间（取第一张图） |
| `地点` | 文字 | 水印中的门店地址（取第一张图） |

## 九、已知局限与建议

| 局限 | 影响 | 建议 |
|------|------|------|
| 所有图片均无正向完整背面照片（Phase 1 无候选） | `是否规范粘贴=0`，`封口贴存在=0` | 要求提交正面完整背面照片 |
| 包装盒检测走 `fallback` | Phase 1 直接排除该图，不参与后续检测 | 人工复核该订单 |
| 严重斜拍（>60°）时 OCR 可能漏字 | 贴纸位置无法确认，`position_valid=-1` | 要求拍摄角度正对盒面 |
| 角度验证以包装盒水平为基准（`box_angle=0`） | 整体大幅倾斜拍摄时基准偏差，角度误判风险升高 | 搭配 Phase 1 的正向完整照片过滤可大幅降低此场景概率 |
| 平铺检测依赖 OCR 识别"Authorized Reseller" | 英文模糊时可能漏判平铺 | 此时 `position_valid` 为 0（位置异常），不会误报合规 |
| `rel_y` 范围 0%~30% 固定 | 不同品类贴法不同（如 Mac 贴在顶部侧面） | 可按 LOB 列配置不同阈值 |
| `STICKER_ANGLE_MAX_DEG` 固定为 10° | 批量生产时不同品类/贴纸形态可能需要不同容忍度 | 可按 LOB 列配置不同阈值 |
