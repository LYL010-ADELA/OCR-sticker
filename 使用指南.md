## 🚀 快速开始

### 1. 启动处理

```bash
cd /home/ubuntu/OCR

# 后台运行
nohup python3 ocr_batch_process_final.py > ocr_final.log 2>&1 &

# 记录进程ID
echo $! > ocr_process.pid
```

### 2. 查看进度

```bash
# 实时查看日志
tail -f ocr_final.log

# 查看已处理行数
wc -l forOCR_results.csv

# 查看最新结果
tail -20 forOCR_results.csv

# 统计有封口贴的数量
grep ",1," forOCR_results.csv | wc -l
```

### 3. 停止处理（如需要）

```bash
# 获取进程ID
cat ocr_process.pid

# 优雅停止
kill $(cat ocr_process.pid)
```

### 4. 恢复处理（如中断）

```bash
# 直接重新运行即可，自动从中断处继续
nohup python3 ocr_batch_process_final.py > ocr_final.log 2>&1 &
```

## 🔧 故障排除

### 问题1：进程被杀死
**原因：** 磁盘空间不足  
**解决：**
```bash
# 检查磁盘空间
df -h

# 清理空间
rm -rf ~/.cache/pip
conda clean --all -y
```

### 问题2：GPU内存不足
**原因：** 其他程序占用GPU  
**解决：**
```bash
# 查看GPU使用情况
nvidia-smi

# 释放GPU
pkill -f python
```

### 问题3：网络超时
**原因：** 图片下载失败  
**解决：** 程序会自动标记为ERROR并继续，无需干预

### 问题4：想查看CSV进度但Excel打开很慢
**解决：**
```bash
# 查看最后100行
tail -100 forOCR_results.csv

# 或者只查看有封口贴的
grep ",1," forOCR_results.csv | tail -50
```
